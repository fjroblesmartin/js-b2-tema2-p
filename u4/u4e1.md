# Tema 2. Trabajo experto con clases

## Unidad 4. Interfaces

## Ejercicio 2.4.1 (Castellano)

JavaScript no proporciona de forma directa mecanismos específicos para usar el concepto de Interfaces en programación, pero contamos con patrones que nos permiten contar con una funcionalidad similar, basándonos en el control sobre la implementación de diferentes métodos por parte de las clases.

En este ejercicio vamos a practicar una técnica que nos permite validar si una clase implementa una interfaz, que definiremos a través de un objeto genérico.


### A. Objeto `IPrintable`

Crear un objeto literal denominado `IPrintable`, con un único método `print` que es una simple función vacía.


### B. Método global `implements`

Añade al prototipo de la clase global `Object` un método `implements` que cumpla las siguientes características:

- Que este método `implements` NO sea iterable / enumerable.
- Que reciba como parámetro un objeto `interf` y compruebe para cada uno de los métodos "iterables" de `interf` que está disponible en el objeto que llame a `implements`.

La respuesta de este método `implements` a de atender a las siguientes casuísticas:

1) Devolver `true` cuando el objeto que llama a `implements` dispone de todos y cada uno de los métodos del objeto `interf` que recibe como parámetro.

2) Devolver `false` en cuanto no disponga de una de esos métodos.


### C. Clase `Message`

Crea una clase `Message` con una propiedad `text`.


### D. Constructor de clase

Implementa el constructor de la clase de manera que reciba como único parámetro un string que se asignará a la propiedad `text`.

Además, el constructor comprobará que implementa la clase `IPrintable` y en caso de que no la implemente por completo, lanzará este error:

```js
throw new Error('ERROR_IMPLEMENTS. This class doesn\'t implements the interface IPrintable.');
```

### E. Método `print`
Implementa en dicha clase un método `print` que muestre por consola el valor de la propiedad `text`.


### F. Exportación de interfaz y clase

Exporta la interfaz `IPrintable` y la clase `Message` para poder ser usada en otros módulos.

# Tema 2. Treball expert amb classes

## Unitat 4. Interfaces

## Exercici 2.4.1 (Català)

JavaScript no proporciona de forma directa mecanismes específics per utilitzar el concepte d'interfícies en programació, però comptem amb patrons que ens permeten comptar amb una funcionalitat similar, basant-nos en el control sobre la implementació de diferents mètodes per part de les classes.

En aquest exercici practicarem una tècnica que ens permet validar si una classe implementa una interfície, que definirem a través d'un objecte genèric.


### A. Objecte `IPrintable`

Crear un objecte literal anomenat `IPrintable`, amb un únic mètode `print` que és una simple funció buida.


### B. Mètode global `implements`

Afegeix al prototip de la classe global `Object` un mètode `implements` que compleixi les següents característiques:

- Que aquest mètode `implements` NO sigui iterable/enumerable.
- Que rebi com a paràmetre un objecte `interf` i comprovi per a cadascun dels mètodes "iterables" d'`interf` que està disponible en l'objecte que anomeni `implements`.

La resposta d'aquest mètode `implements` ha d'atendre les casuístiques següents:

1) Tornar `true` quan l'objecte que crida a `implements` disposa de tots i cadascun dels mètodes de l'objecte `interf` que rep com a paràmetre.

2) Tornar `false` tan bon punt no disposi d'un d'aquests mètodes.


### C. Classe `Message`

Crea una classe `Message` amb una propietat `text`.


### D. Constructor de classe

Implementa el constructor de la classe de manera que rebi com a únic paràmetre un string que s'assignarà a la propietat text.

A més, el constructor comprovarà que implementa la classe IPrintable i en cas que no la implementi per complet, llançarà aquest error:

```js
throw new Error('ERROR_IMPLEMENTS. This class doesn\'t implements the interface IPrintable.');
```

### E. Mètode `print`
Implementa en aquesta classe un mètode `print` que mostri per consola el valor de la propietat `text`.


### F. Exportació d'interfície i classe

Exporta la interfície `IPrintable` i la classe `Message` per poder ser utilitzada en altres mòduls.
